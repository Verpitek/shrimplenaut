How to use Shrimplenaut?

/packages - this is where all the packages are retrieved, by default the packages start from the oldest to newest, 10 items per page, starting at page 1

You can apply custom parameters for easier filtering of results

?page - sets the page, example usage: shrimplenaut.verpitek.com/packages?page=2

?per_page - sets the amount of items to return per page, example usage: shrimplenaut.verpitek.com/packages?per_page=5

the default is set to 10, and the max is 100

?tag - lets you filter packages by a tag, example usage: shrimplenaut.verpitek.com/packages?tag=Economy

possible selections: Economy, Library, Management, Minigame, Chat, Cursed, Misc, Utility and Tools, Game Mechanics

?id - lets you find a package by an id, but other filters are ignored: shrimplenaut.verpitek.com/packages?id=2

?name - lets you find a package by its name, but other filters are ignored: shrimplenaut.verpitek.com/packages?name=Economy Pilot

?license - lets you find a package by its license, but other filters are ignored: shrimplenaut.verpitek.com/packages?license=MIT

possible selections: Economy, Library, Management, Minigame, Chat, Cursed, Misc, Utility and Tools, Game Mechanics

?server_software - lets you find a package by its server software, but other filters are ignored: shrimplenaut.verpitek.com/packages?server_software=endstone

possible selections: endstone, leviLamina

?project_type - lets you find a package by project type, but other filters are ignored: shrimplenaut.verpitek.com/packages?project_type=plugin

possible selections: plugin, addon, vAddon, structure

?search - allows you to do soundalike searching by name, but other filters are ignored: shrimplenaut.verpitek.com/packages?search=Ecnmy pilt <- would return the packages closest to that query

you can chain parameters that can be chained by using & after each one

Full example: shrimplenaut.verpitek.com/packages?page=2&per_page=5&tag=Economy
